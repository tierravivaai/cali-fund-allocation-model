# Stewardship Pool Volume Analysis

## Purpose

This folder contains reproducible data for Tables E1 and E2 from the Option D revision draft, restructured for clarity. The original Table E1 packed three values (IUSAF|TSAC|SOSAC) into each cell separated by pipes, making it very hard to read. The restructured version separates IUSAF from the Stewardship Pool (TSAC+SOSAC combined), which is both cleaner and sufficient — TSAC and SOSAC individual amounts can be derived from the percentages in the header row.

## Scenario Parameters

- 142 eligible Parties (high-income countries excluded, except SIDS)
- Fund sizes: $50m, $200m, $500m, $1bn, $5bn, $10bn
- SOSAC fixed at 3%
- State/IPLC split: 50/50
- UN scale mode: band inversion

## Reading the Original Table E1 Format

The original draft used pipe-separated triple values (e.g., `955.0|15.0|30.0`) representing IUSAF|TSAC|SOSAC. These can be reconstructed from the restructured tables:

- **TSAC** = Pool × (TSAC% / Pool%)
- **SOSAC** = Pool × (SOSAC% / Pool%)
- Or simply: TSAC = Pool at the given TSAC%, SOSAC = Pool at the given SOSAC%

For example, at the Gini-minimum and $1bn:
- Pool = $55.0m
- TSAC = 55.0 × (2.5/5.5) = $25.0m
- SOSAC = 55.0 × (3.0/5.5) = $30.0m ✓

## Reproducibility

All outputs can be regenerated by running:

```bash
python3 band-analysis/stewardship-pool/stewardship_pool_analysis.py
```

The component amounts are deterministic from the percentage splits and fund sizes — no calculator run is needed since IUSAF = α × Fund, TSAC = β × Fund, SOSAC = γ × Fund.
